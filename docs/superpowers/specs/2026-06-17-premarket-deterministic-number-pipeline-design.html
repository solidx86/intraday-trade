Deterministic Number Pipeline for the Pre-Market Briefing

Design spec · 2026-06-17 · skill: premarket-briefing-skill · scope: numbers only (news harvest unchanged)

Draft — awaiting user review

Every dollar price and percent move in the briefing is presently produced by the model reading a semi-structured quote page and transcribing what it sees. On 2026-06-17 that pipeline failed three ways in a single run. This redesign removes the model's hands from the number pipeline: a script fetches numbers from a structured JSON source, a validator gates the finished briefing against that machine output, and the model only narrates.

The defects (2026-06-17 incident)

#DefectRoot cause
1Tuesday's close-to-close moves printed as Wednesday "pre-market" (e.g. AAPL "+0.95%", GOOGL "+1.06%" were the prior session, not pre-market).finviz quote.ashx free endpoint serves the last completed regular session's close, but Step 3.5 treats it as the live "current/pre-mkt" anchor.
2No check that a quoted number was actually a pre-market tick versus a prior close.The anchor table has a "Current/pre-mkt" column but nothing verifies the figure carries a pre-market timestamp/label.
3A WebSearch snippet's number was printed, and it was the wrong share class — GOOG (Class C) used for GOOGL (Class A, the watchlist symbol).No exact-symbol/share-class guard; no rule forbidding a search snippet from being the printed price.
The principle being corrected. "Deterministic" cannot mean the values are identical across runs — the market moves between fetches. It means the extraction, labeling, and verification are mechanical (code), not model judgment, and the finished artifact is validated against the machine output. The model can't invent a number at the source (script owns extraction) and can't alter or fabricate one in the write-up (validator gates the saved file). Both, or the guarantee leaks. Determinism guarantees faithful reproduction of the feed, not that the feed is correct.

The unlock — endpoints verified on 2026-06-17

Both candidate sources were probed live (stdlib urllib, with a User-Agent) before this spec was written.

CNBC quote.htm — PRIMARY (verified)

https://quote.cnbc.com/quote-html-webservice/quote.htm?symbols=<list>&requestMethod=itv&exthrs=1&output=json — no auth, batch multi-symbol, returns ITVQuoteResult.ITVQuote[]. Verified return for AAPL | .SPX | .VIX | @CL.1 | .GDAXI | .N225 | .KS11 (stock, index, vol, future, European + Asian indices incl. KOSPI). Each quote carries previous_day_closing, last, change_pct, curmktstatus, and — decisively — an ExtendedMktQuote block:

AAPL  previous_day_closing=299.24  curmktstatus=REG_MKT
  ExtendedMktQuote: { type:"PRE_MKT", last:"300.92", change_pct:"+0.56%",
                      last_timedate:"9:30 AM EDT", last_time:"2026-06-17T09:30:00-0400" }
CNBC hands us the three gate inputs as fields to read, not things to infer: quote-type (ExtendedMktQuote.type = PRE_MKT), an ISO timestamp (last_time), and the prior close (previous_day_closing) — in one no-auth batch call, from the same data behind the four CNBC market pages the briefing already references.

Yahoo v7 quote — verified, but NOT used (dropped for simplicity)

Decision: Yahoo was probed and works, but the final design is CNBC-only — CNBC already supplies typed pre-market, timestamp, prior close, and exact-symbol echo, so the Yahoo cross-check and fallback (and the crumb auth dance) are dropped. The note below is retained as the record of what was verified, should a fallback ever be reinstated.

query1.finance.yahoo.com/v7/finance/quote?symbols=<list> returns 401 bare, but the crumb+cookie flow runs in stdlib (seed cookie at fc.yahoo.com → crumb at /v1/test/getcrumb → quote). Verified return incl. ES=F, ^VIX, ^TNX, EURUSD=X, ^GDAXI, ^KS11, XLK with correct regularMarketPreviousClose (AAPL 299.24). Fields preMarketPrice / preMarketChangePercent / marketState exist; they read None / REGULAR only because the probe ran after the 09:30 ET open. v8-chart (no-auth) is a weak last-resort — no clean pre-market or prior close.

Banned as a price finviz quote.ashx (free) — its price is the prior session's close. Demoted to headlines / analyst actions / earnings dates only. A WebSearch snippet may never be a printed price/% (it can inform the story).

Goals

  • Replace model-read numbers with a scripted fetch from CNBC's JSON (tape + per-ticker). CNBC-only — no Yahoo.
  • Make quote-type (PRE-MKT / PRIOR-CLOSE / N/A), timestamp, and exact-symbol matching mechanical.
  • Add a validator that fails the briefing if any printed number can't be tied to the machine ledger.
  • Retire the browser as a number source (it stays for news/themes) — also eliminating the "snapshot didn't render" failure hit twice on 2026-06-17.
  • Keep the briefing's structure, voice, and 7 sections intact. This changes how numbers arrive, not the output format — beyond per-number session tags.

Design

1. Source reclassification

SourceRole
CNBC quote.htm JSONPrimary for every number — tape (futures, VIX/VXN, sectors, commodities, US yields, FX, all global indices) and per-ticker.

2. The fetch script — scripts/fetch_market_data.py (new)

Stdlib only (urllib, json, re) so it runs under the PEP-668-locked system python3 with no pip. CNBC accepts a batched symbol list in one request, so no threadpool and no second-endpoint auth are needed. Invoked by the agent via one Bash call at Step 3.5; the agent consumes stdout verbatim and never reads a price off a page.

  • Inputs: the per-ticker watchlist (read from watchlist.md) plus a fixed tape symbol map (CNBC symbology — e.g. @ES.1 @NQ.1 @YM.1 .VIX .VXN US10Y @CL.1 @GC.1 @NG.1 EUR= JPY= .N225 .HSI .SSEC .KS11 .GDAXI .FTSE .STOXX50E, sectors via index or SPDR ETF). The complete map is finalized at build time; any symbol that fails to resolve emits N/A, never a guess.
  • Concurrency: batch request first (CNBC accepts a |-joined list in one call). If per-symbol requests are needed (fallback/Yahoo), a ThreadPoolExecutor(max_workers≈8) inside the script. Output is sorted by input order so the ledger is byte-identical regardless of completion order.
  • Quote-type derivation (mechanical): PRE-MKT when ExtendedMktQuote.type == "PRE_MKT" with a today-dated last_time; else PRIOR-CLOSE; N/A on fetch failure or symbol-echo mismatch.
  • Symbol/class gate: the response symbol must equal the requested symbol; a mismatch (e.g. GOOG≠GOOGL) → rejected to N/A.
  • No cross-check, no Yahoo: CNBC is the single source; any miss is N/A (never guessed). The symbol-echo check (response symbol must equal the request) is what prevents a GOOG-for-GOOGL substitution, and it lives in the CNBC parser.
  • Outputs: a JSON sidecar daily/<US-date>/.quote-ledger.json (machine source of truth for the validator) and a human-readable markdown ledger to stdout (what the model pastes into scratch).
  • Failure = N/A, never a guess. Broad outage → the briefing prints N/A and says so.

3. The unified ledger (replaces the hand-built anchor + tape tables)

One row per number, tape and ticker alike:

symbol(exact) | prior_close | last/pre_mkt | chg_pct | timestamp(ET) | source | quote_type

Quote-type is one of PRE-MKT / PRIOR-CLOSE / N/A, set by the script.

4. Three hard gates

  1. Pre-market gate: a number prints as "today's move" only if quote_type == PRE-MKT with a timestamp. A PRIOR-CLOSE figure is tagged (prior close) in the body or marked N/A — never dressed as today's move.
  2. Symbol/class gate: exact watchlist symbol, response-echo verified.
  3. Provenance gate: the printed number traces to the ledger (CNBC/Yahoo structured fetch), never a search snippet or model memory.

5. Output convention (template)

Every per-ticker and tape number in the body carries a session tag: (pre-mkt) / (prior close) / (N/A — reason) — the convention adopted by hand in the 2026-06-17 briefing. The reader always knows which session a number reflects.

Grouped tags (readability + machine-checkable): one tag may scope a list of numbers within a single bullet/line (e.g. "Semis bouncing (pre-mkt): AMD +2.67%, MU +3.26%, NVDA +0.51%"). The validator scopes a tag to its bullet — a group tag covers every number listed until the next tag — so grouping stays parseable. Every number must fall under some tag; an untagged number fails validation.

6. The validator — extend evals/validators/validate_briefing.py

Parses the saved premarket.md, extracts every $price and ±% with its ticker and session tag, and cross-checks against .quote-ledger.json. Fails when:

  • a (pre-mkt)-tagged number has no PRE-MKT ledger row;
  • a printed value doesn't match the ledger (within rounding);
  • a ticker isn't in the watchlist / exact symbol;
  • a number carries no session tag.

Enforcement is two-layered:

  • Step-5 self-check (first line): SKILL.md instructs the agent to run the validator after writing premarket.md and fix/N/A any flagged number before presenting. Keeps the common path clean.
  • Hard hook (backstop — cannot be skipped): a PostToolUse hook in .claude/settings.json, matched to Write/Edit on daily/*/premarket.md, runs the validator automatically and returns a blocking decision on failure regardless of agent compliance. The hook reads the just-written file + the sibling .quote-ledger.json. Rule: no ledger present → block (forces the deterministic fetch path); a ledger present with all-N/A rows is allowed (an honest outage isn't an error). The hook early-exits silently for any non-premarket write, so it is inert for all other work — and for anyone cloning the public repo.

The verdict is mechanical in both layers; the hook is what makes "every invocation" literally true. CI: tests/test_quote_ledger.py covers the validator logic and the hook's stdin/path wrapper (given a Write-event JSON → correct block/pass) against fixtures, so the suite stays offline (mirrors tests/test_premarket_validator.py). The settings.json registration is applied via the update-config mechanism.

Testing

  • tests/test_quote_ledger.py — validator logic against fixture briefing + fixture ledger (pass case; each fail case: untagged number, pre-mkt-without-row, value mismatch, unknown symbol).
  • The fetch script itself is exercised with a recorded-JSON fixture (parse → ledger), so the test suite stays offline/deterministic (no live network in CI).
  • Existing tests/test_premarket*.py reviewed for any finviz-as-price assumption.

Honest constraints & risks

  • Endpoint reliability: CNBC/Yahoo are unofficial JSON; either can change shape or rate-limit. Mitigation: dual source + N/A on failure + browser last-resort; the validator never blocks on missing data, only on wrong data.
  • Symbol map drift: CNBC symbology for the tape must be enumerated and maintained; an unverified symbol degrades to N/A (loud), not a guess (silent).
  • Pre-market vs open timing: if the skill runs after 09:30 ET the script returns REG_MKT last prints, correctly tagged (prior close)/regular — not mislabeled pre-market.
  • Determinism scope: pipeline, not values (restated from the principle box).

Out of scope (decided)

  • News via RSS. Yahoo per-symbol RSS and CNBC Top-News RSS were verified working, but news is prose the model summarizes (no number to hallucinate; the validator doesn't gate it). The browser news harvest (Step 3b) is unchanged. Not pursued.
  • No change to the briefing's sections, voice, regime read, or spillover logic.
