Browser-Sourced Market Tape for the Pre-Market Briefing

Date: 2026-06-17  ·  Skill: premarket-briefing-skill  ·  Branch: feat/premarket-browser-tape

Approved design — implemented

Problem

The briefing gathers data with WebFetch + WebSearch. The most valuable structured-data sources are exactly the ones WebFetch can't reach: cnbc.com returns 403, and reuters.com, investing.com, and forexfactory.com sit behind Cloudflare/bot walls. As a result the economic calendar, earnings actuals, and Global Spillover silently degrade.

And, wearing the intraday-trader hat, the briefing has never carried the single most important pre-bell numbers: index futures / implied open, the VIX/VXN volatility regime, the sector rotation tape, and commodities — precisely because that data was hard to fetch.

The unlock

The Playwright headless browser loads cnbc.com/markets/pre-markets/ cleanly. One load yields futures + fair-value implied open, all Treasury tenors, FX, commodities, VIX/VXN, sector performance, and Asia/Europe indices — enough to fill four new trader lines and most of Global Spillover and the dollar/yields regime read, from a single fetch.

Token discipline
browser_navigate writes the full ~100k-char snapshot to a .playwright-mcp/page-*.yml file and returns only a tiny confirmation. The recipe greps the saved file for the ~15 data rows it needs — the raw page never enters context. Calling browser_snapshot without a filename (which dumps the page into context) is explicitly forbidden.

Goals

  • Wire the browser in as the primary path for the bot-walled structured sources, consolidated through one CNBC "tape" dashboard load.
  • Add four enforced tape lines to section 1.1: Futures, Volatility, Sector tape, Commodities.
  • Keep the 7-section output structure otherwise intact — reliability + key additions, not a redesign.
  • Anti-fabrication: every tape number is a grepped figure or an explicit N/A — reason; unify that rule with the existing per-ticker price discipline.

Design

1. New gather phase — Phase C: market tape (browser)

A new sub-step (SKILL.md Step 3a) that runs first among the gather phases. It navigates CNBC pre-markets, greps the saved snapshot, and builds a Tape Table — a mandatory internal scratch artifact, sibling to the Step 1 date-anchor and Step 3.5 price-anchor tables. The Tape Table is the anchor for every tape number: nothing reaches the body unless its exact figure is in the grep output.

Phase A (news harvest) and Phase B (targeted lookups) keep their identities; the phase letters denote identity, not running order. Execution order is C → A → B.

Tape table (CNBC pre-markets, grepped HH:MM MYT):
  Futures:    ES <fut> (±%)  · NQ <fut> (±%)  · Dow <fut> (±%)  [implied-open Δ]
  Volatility: VIX <lvl> (±%) · VXN <lvl> (±%)
  Sectors:    <leader> +x% … <laggard> −y%
  Commodities:WTI <lvl> · Gold <lvl> · NatGas <lvl>
  Yields/FX:  US10Y <%> · DXY <dir> · EUR/USD <lvl> · USD/JPY <lvl>
  Global:     Nikkei <±%> · HSI <±%> · DAX <±%> · FTSE <±%> · STOXX <±%>

2. Four new section-1.1 tape lines

Inserted between the three news bullets and the existing Market mood line:

**Futures (implied open):** ES +0.25% · NQ +0.46% · Dow +0.14% — firm into the open.
**Volatility:** VIX 16.4 (+1.3%) · VXN 27.0 (+4.0%) — **elevated**
**Sector tape:** Financials +1.5% lead · Tech −2.3% lag — defensive rotation.
**Commodities:** WTI $76.1 · Gold $4,361 · NatGas $3.25

VIX tag scale: <15 calm · 15–20 normal · 20–25 elevated · >25 high. These lines also feed the regime read (US10Y direction) and most of Global Spillover (Asia/Europe indices, FX) — routed from the Tape Table instead of separate searches.

3. Unified N/A — reason failure rule

The tape fields are number-or-nothing: a real grepped figure, or N/A — <one-clause reason> stating why (page layout changed, module didn't load). No directional prose substitutes for a number. The same rule is applied to the existing Step 3.5 per-ticker price discipline (replacing its old "write directional language only" instruction), so the whole briefing fails the same honest, auditable way.

4. Source tiering

SourcePath
CNBC, Reuters, investing.com, forexfactorybrowser-primary (skip the wasted WebFetch 403)
finviz (news, quote, groups, calendar)WebFetch-first, browser fallback; quote.ashx stays the per-ticker price anchor
marketwatch.comWebFetch-first, browser fallback

Graceful degradation escalates by tier: WebFetch → browser → WebSearch → N/A — reason.

5. Validator — four new checks

  • check_tape_lines in evals/validators/validate_briefing.py verifies each of the four bold tape labels is present in section 1.1.
  • N/A — reason satisfies the check — it asserts the field showed up honestly, not that a number was found.
  • Wired into run_all_checks (now 11 checks); module docstring updated.

6. Tests (bite-style)

New unit tests in tests/test_premarket_validator.py prove each tape check actually bites: all-present passes; removing any one line fails its check; an N/A — reason line passes; run_all_checks emits the four named results.

7. Sample data + eval rubric

Both tracked briefings gain the four tape lines in 1.1 (synthetic but plausible, consistent with each day's narrative). One rubric criterion added to evals.json.
